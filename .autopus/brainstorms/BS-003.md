## 추천 방향

**"Interactive Pane + Hook 자동주입 + 멀티턴 핑퐁"** (R6 완성 + 멀티라운드 통합)

핵심 흐름:
```
[선행] R6 hook 자동주입
  ├─ claude:   .claude/settings.json → hooks.Stop에 hook-claude-stop.sh 등록
  ├─ gemini:   .gemini/settings.json → hooks.AfterAgent에 hook-gemini-afteragent.sh 등록
  └─ opencode: opencode.json → experimental.plugins에 hook-opencode-complete.ts 등록

[본체] Interactive 멀티턴 핑퐁
  Round 1: pane 3개 열기 → CLI 진입 → 프롬프트 전송 → hook done 파일로 완료 감지 → result.json 파싱
  Round 2: 같은 pane에 반박 프롬프트 전송 → hook done 파일 감지 → result.json 파싱
  Round 3: judge pane에 통합 요청 → 최종 결과
  정리: pane 닫기
```

이유:
1. hook 스크립트 3개가 이미 `content/hooks/`에 구현됨 — 설치 자동화만 필요
2. Interactive 모드의 결과 수집 문제가 hook으로 해결됨 (ReadScreen/idle 불필요)
3. pane을 유지한 채 멀티턴 → 사용자가 실시간으로 토론 관찰 가능
4. 가장 만족스러운 UX — "진짜 핑퐁" 경험
5. Sentinel fallback은 hook 미설치 환경용으로 자동 전환 (R8 graceful degradation 이미 구현)

구현 순서:
1. **R6 hook 자동주입** — `auto init` 시 3개 CLI 설정 파일에 hook 등록
2. **멀티턴 라운드 루프** — interactive.go에 라운드 루프 추가, 라운드별 hook done 파일 감시
3. **rebuttal 프롬프트 생성** — debate.go의 buildRebuttalPrompt()를 interactive에서 호출
4. **--rounds N 플래그** — CLI에서 라운드 수 지정 가능
