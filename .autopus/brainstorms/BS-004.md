### 테스트 관찰

2. OpenCode 프롬프트 주입 실패 — 어댑터 계층 필요
3. 프로바이더별 응답 완료 대기 메커니즘 부족
4. UI 크롬(터미널 렌더링)이 응답 콘텐츠와 분리되지 않음

## 추천 방향

테스트 결과, 멀티턴 토론 개선의 **가장 시급한 블로커**는 다음 3가지:

1. **프로바이더 응답 캡처 품질** (Rank 1, 2) — 응답이 완료되기 전에 캡처하거나, UI 크롬과 실질 콘텐츠가 혼합됨. 응답 완료 감지와 콘텐츠 파싱 강화가 선행되어야 함.

2. **프로바이더별 프롬프트 어댑터** (Rank 3) — OpenCode처럼 프롬프트가 제대로 주입되지 않는 프로바이더 존재. CLI 인터페이스 차이를 추상화하는 어댑터가 필요.

3. **멀티턴 프로토콜** (Rank 4) — 현재는 단일 라운드만 실행. Round N의 출력을 파싱하여 Round N+1의 "상대 의견 반박" 프롬프트로 주입하는 명시적 프로토콜 구현 필요.
