## 원본 아이디어
- What: `--multi` 플래그 시 자동으로 터미널 창을 분할하고, 각 에이전트(planner, executor, tester, reviewer)의 실행을 별도 패인에서 시각적으로 보여주는 기능
- Why: 현재 서브에이전트 파이프라인은 "보이지 않는" 실행 — 사용자가 진행 상황을 실시간으로 확인 불가. cmux/dmux/NTM 등 경쟁 도구가 이미 이 패턴을 구현 중
- Who: autopus-adk 사용자 (AI 에이전트로 소프트웨어를 개발하는 개발자)
- When: cmux(manaflow-ai)가 이미 성숙하고 Claude Code Agent Teams도 split pane 지원하는 시점

## 리서치 결과

### cmux (manaflow-ai/cmux) — macOS 네이티브 터미널
- Swift + AppKit, libghostty 렌더링
- Socket API로 프로그래밍 방식 패인 분할 (`cmux workspace create`, `cmux pane split`)
- 알림 시스템: `cmux notify` CLI — 에이전트 훅 통합 가능
- 내장 브라우저: 접근성 트리 스냅샷, 폼 입력, JS 실행
- AGPL-3.0, `brew install --cask cmux`

### 관련 도구 생태계
| 도구 | 설명 | 특징 |
|------|------|------|
| **dmux** | tmux 패인 매니저 | git worktree + 독립 브랜치 per 패인 |
| **NTM** | Named Tmux Manager | 멀티 에이전트 타일, 브로드캐스트, TUI 대시보드 |
| **claude-squad** | AI 에이전트 관리 | Claude/Codex/OpenCode/Amp 멀티 관리 |
| **Claude Code Agent Teams** | 내장 split pane | tmux/iTerm2 자동 분할, 공유 태스크 리스트 |

### Claude Code Agent Teams 핵심 사항
- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 필요
- display mode: `in-process` | `split panes` (tmux) | `auto`
- 세션당 1개 팀, 중첩 불가, 권장 3-5 teammate

## 프로바이더별 발산 결과

### 아이디어 A: Terminal Adapter Layer in Go Binary
Go 바이너리에 `pkg/terminal/` 패키지로 Adapter interface 추가.
구현체: TmuxAdapter, CmuxAdapter, ITermAdapter, InProcessAdapter.
자동 감지로 환경에 맞는 어댑터 선택.

### 아이디어 B: cmux-first Integration
cmux Socket API만 타겟. Go 바이너리가 cmux CLI/Socket으로 워크스페이스 생성, 패인 분할.
macOS 전용이지만 최적화된 AI 에이전트 경험.

### 아이디어 C: Claude Code Agent Teams 확장
기존 `--team` 모드와 `--multi` 결합. Agent Teams의 split pane 기능을 autopus pipeline에 통합.
바이너리 변경 최소, 이미 검증된 패턴.

### 아이디어 D: Hybrid — Agent Teams + Terminal Dashboard
Agent Teams + NTM 스타일 TUI 대시보드. 최고의 시각 경험이지만 복잡도 극대.

### 아이디어 E: Skill-Based Multi-Agent Orchestration
스킬 레벨에서 tmux 분할. Bash tool로 tmux 명령 실행.
바이너리 변경 없이 즉시 적용 가능.

## 추천 방향

**cmux-first, tmux-fallback 바이너리 통합 (B → G → F)**

### v1 MVP — cmux Socket API Integration
Go 바이너리에 `pkg/terminal/cmux.go` 추가:
- `auto go --multi` → cmux Socket API로 워크스페이스 생성 + 패인 분할
- 각 패인에서 `auto agent run <task-id>` 서브커맨드 호출
- `cmux notify` 통합: 에이전트 주의 필요 시 파란 링 알림
- cmux 내장 브라우저 활용: 프론트엔드 에이전트가 dev 서버 직접 조작 가능
- cmux 없으면 → v1.1 tmux fallback으로 자동 전환
- **작업량**: `pkg/terminal/cmux.go` + `cmd/auto/agent_run.go` + flag 파싱
- **배포**: `auto update`로 모든 사용자 적용

### v1.1 — tmux Fallback (Idea G)
- cmux 미설치 환경 대응
- `pkg/terminal/tmux.go` — tmux CLI 래핑
- 둘 다 없으면 graceful error: `"--multi requires cmux or tmux"`

### v2 — Agent Teams 연동 (Idea C)
- `--multi --team` 조합 시 Agent Teams display mode 자동 설정
- autopus pipeline Phase와 Agent Teams 역할 매핑

### v3 — Unified Terminal Controller (Idea F)
- `pkg/terminal/terminal.go` — Terminal interface 추상화
- `DetectTerminal()` 자동 감지: cmux > tmux > iTerm2 > in-process (우선순위)
- 어댑터 패턴으로 신규 터미널 확장 용이

### 차별점 (vs dmux/NTM/claude-squad)
- **dmux/NTM/claude-squad** = "터미널 관리 도구" (범용 에이전트 패인 매니저)
- **autopus --multi** = "파이프라인 네이티브" (Phase 전환 시 패인 레이아웃 동적 변경)
- **cmux 네이티브**: Socket API + notify + 내장 브라우저를 파이프라인에 직접 활용
- Phase 1: 1패인(planner) → Phase 2: N패인(executor×N) → Phase 3: 1+브라우저(tester+verify) → Phase 4: 2패인(reviewer+auditor)
- worktree isolation + GC suppression + 머지 충돌 처리가 바이너리에 내장
