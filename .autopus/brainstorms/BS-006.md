## 원본 아이디어
- What: --team 모드에서 각 에이전트(lead/builder/tester/guardian)를 cmux pane으로 시각화하여 실시간 모니터링
- Why: 현재 Agent Teams는 SendMessage 텍스트로만 상태 확인 가능. SPEC-ORCH-001에서 orchestra cmux 연동을 구현했으므로 동일 인프라(pkg/terminal) 확장
- Who: Autopus ADK 사용자 (개발자) — --team 멀티에이전트 파이프라인 실행 시
- When: SPEC-ORCH-001 완료 후 자연스러운 확장

## 핵심 제약
- Agent Teams / 서브에이전트는 Claude Code 내부 API (Agent tool, SendMessage)로 동작
- 외부 CLI처럼 cmux pane에서 직접 실행할 수 없음 (orchestra와 근본적 차이)
- 따라서 "실행 시각화"가 아닌 "상태 모니터링 대시보드" 접근 필요

## 브레인스토밍 결과 (SCAMPER)

### 아이디어 후보

1. **Full Pane Monitor**: 에이전트별 pane(tail -f 로그) + 대시보드 pane
2. **Unified Log + Dashboard**: 통합 로그 1개 + 대시보드 1개 (2 pane)
3. **Hook-driven Monitor**: settings.json hooks로 에이전트 활동 자동 감지/표시
4. **TUI Dashboard Only**: bubbletea 전체 화면 대시보드 (cmux pane 분할 없음)
5. **Hybrid**: 대시보드 pane + 선택적 에이전트 pane (사용자가 포커스 선택)

### 구현 메커니즘 (공통)
- 에이전트 프롬프트에 "진행 상황을 {logfile}에 기록하라" 지시 추가
- cmux pane에서 `tail -f {logfile}` 실행
- 메인 pane에 파이프라인 진행 상태 렌더링
- 파이프라인 완료 시 pane 정리 (cleanupPanes 재활용)

## 추천 방향

**Unified Log + Dashboard (Top 1)**가 최선. 이유:
- 2 pane만으로 구현 단순
- 에이전트가 공유 로그 파일에 구조화된 메시지(`[timestamp] [agent] [phase] message`) 기록
- 대시보드 pane에 TaskList 기반 진행 상태 표시
- 서브에이전트 모드에서도 동일하게 적용 가능 (--team 전용 아님)

**Hybrid (Top 2)**는 후속 확장으로 적합:
- Unified Log 기반 위에 "특정 에이전트 포커스" 기능 추가
- `auto team watch {agent-name}` 커맨드로 선택적 pane 생성
